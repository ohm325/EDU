PROJECT PROPOSAL & ARCHITECTURE

โครงการพัฒนาระบบ Standalone iPad POS

สถาปัตยกรรมระบบขายหน้าร้านแบบเอกเทศ (Decoupled Architecture) รองรับการทำงานออฟไลน์ ผูกสิทธิ์ระดับอุปกรณ์ (Device Binding) และเชื่อมต่อ ERP ผ่านตัวกลาง WELLPORTAL

เป้าหมายของโครงการ (Project Goals)

ความปลอดภัยสูงสุด

ตรวจสอบสิทธิ์ผ่าน Active Directory (AD) และบังคับผูก User ติดกับ Hardware ID ของ iPad (เครื่องใครเครื่องมัน ห้ามสลับเครื่อง)

ทำงานได้แม้ออฟไลน์

แอปพลิเคชันบน iPad สามารถเก็บข้อมูลสินค้าและราคาไว้ในเครื่อง ทำรายการขายได้ลื่นไหลแม้ไม่มีสัญญาณอินเทอร์เน็ต

ลดภาระ ERP

ใช้ WELLPORTAL เป็นประตูด่านหน้า (Gateway) รับข้อมูล กรองความถูกต้อง ก่อนส่งไปสร้าง Sale Order ที่ระบบหลัก (ERP)

ขอบเขตการพัฒนาระบบ (Scope of Work)

1. ส่วนบริหารจัดการหลังบ้าน (POS Admin & API Backend)

.NET 8 / SQL Server
  • Authentication & Security: สร้าง API ตรวจสอบสิทธิ์ผ่าน OpenAPI (AD) และพัฒนาระบบ Device Binding (สร้าง/ตรวจสอบ UUID ฝั่งเซิร์ฟเวอร์)
  • Management Portal: หน้าเว็บสำหรับผู้ดูแลระบบ เพื่อจัดการสิทธิ์ (Role), ดูประวัติการล็อกเครื่อง, ฟังก์ชัน 'Reset Device', และดู Log รายการขาย
  • Data Sync API: สร้าง Job Schedule ดึง Master Data (ลูกค้า, สินค้า, ตารางราคา, Onhand) จาก ERP มาพักไว้ที่ฐานข้อมูล POS

2. ส่วนแอปพลิเคชันพนักงานขาย (iPad POS Frontend)

Touch-First PWA
  • UI/UX Design: ออกแบบหน้าจอสำหรับ iPad โดยเฉพาะ (ปุ่มขนาดใหญ่, ไร้ Hover, เลย์เอาต์แบ่งครึ่งจอ ซ้าย: เลือกสินค้า ขวา: ตะกร้าและสรุปยอด)
  • Offline-Capable Logic: โหลด Master Data มาเก็บใน Local Storage ของเบราว์เซอร์ เพื่อให้ระบบคำนวณส่วนลดและเช็คสต็อกได้ทันทีโดยไม่ต้องรอโหลดจากเน็ต
  • Draft & Sync: กรณีไม่มีอินเทอร์เน็ต สามารถบันทึกบิลไว้ในเครื่อง (Save Draft) และกดปุ่ม Sync เมื่อมีสัญญาณ เพื่อส่งข้อมูลทั้งหมดเข้าสู่ระบบ

3. ส่วนเชื่อมต่อระบบหลัก (Integration Layer)

JSON API
  • Payload Packaging: iPad นำข้อมูลบิลขายมาแพ็กเป็นโครงสร้าง JSON เพื่อส่งผ่าน API เข้าสู่เซิร์ฟเวอร์
  • Gateway to ERP: WELLPORTAL รับข้อมูล, ตรวจสอบความถูกต้อง (Validation), ส่งต่อให้ ERP สร้าง Sale Order, และส่งเลขที่เอกสาร (SO No.) คืนกลับไปยัง iPad

Workflow การใช้งานของพนักงานขาย (User Journey)

1

เริ่มวันทำงาน: เข้าสู่ระบบ (Login)

เซลส์เปิดแอปบน iPad เข้าสู่ระบบด้วยรหัสพนักงาน (AD) ระบบจะตรวจสอบว่าเครื่องนี้คือเครื่องประจำตัวหรือไม่ หากถูกต้องจึงอนุญาตให้เข้าใช้งาน

2

อัปเดตข้อมูล (Data Sync)

ระบบดาวน์โหลดข้อมูลลูกค้า สินค้าอะไหล่รถบรรทุก และราคาอัปเดตล่าสุดจาก ERP มาเก็บไว้ในเครื่อง iPad เตรียมพร้อมสำหรับการออกไปพบลูกค้า

3

พบลูกค้า: ทำรายการขาย (Sales)

ค้นหาชื่อลูกค้า เลือกสินค้าลงตะกร้า ระบบคำนวณส่วนลดและเช็คสต็อกล่วงหน้าโดยใช้ข้อมูลในเครื่อง (ทำงานได้ 100% แม้ไม่มีอินเทอร์เน็ต)

4

ปิดการขาย: ส่งคำสั่งซื้อ (Checkout)

กดยืนยันบิล หากมีอินเทอร์เน็ตระบบจะส่งเข้า ERP ทันทีเพื่อสร้าง Sale Order หากไม่มีสัญญาณ บิลจะถูกพักไว้ (Save Draft) และจะส่งเข้าสู่ระบบอัตโนมัติเมื่อกลับมามีสัญญาณ
