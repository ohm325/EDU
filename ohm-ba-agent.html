AI Agent

ยกระดับการเก็บความต้องการของระบบ (Requirement) เปลี่ยนคำพูดทั่วไป ให้กลายเป็น "พิมพ์เขียวดิจิทัล" ที่พร้อมส่งให้ AI เขียนโปรแกรมต่อได้ทันที

นำเสนอโดย: โอม (Chief Architect)
เป้าหมาย: ลดความซ้ำซ้อนของการสื่อสารระหว่างคนและเครื่องจักร

ทำไมเราถึงต้องมีผู้ช่วย AI? (The Pain Points)

ปัญหาคลาสสิกของการพัฒนาระบบ ไม่ใช่การเขียนโค้ด แต่คือ "การสื่อสาร"

ข้อมูลกระจัดกระจาย

เวลาประชุม ผู้ใช้มักเล่าความต้องการผสมกันไปมา ขาดลำดับขั้นตอน ทำให้ BA (คน) ต้องใช้พลังงานมหาศาลในการปะติดปะต่อเรื่องราว

วาระแทรก / นอกเรื่อง

ในคำพูด 100% มักจะมี "เรื่องบ่น" หรือ "ไอเดียอนาคต (ที่ยังไม่ทำตอนนี้)" ปะปนมาด้วย ทำให้เสียเวลาแยกแยะและมักเกิดปัญหาขอบเขตงานบานปลาย

คอขวดที่การทำเอกสาร

ทีมงานต้องเสียเวลาหลายชั่วโมงในการฟังเทปประชุมซ้ำๆ และพิมพ์เอกสาร (Backlog) ให้โปรแกรมเมอร์อ่าน ซึ่งมักจะเกิดการตกหล่น

วิธีแก้ปัญหา: "ตะแกรงร่อนความต้องการด้วย AI"

เราจะใช้ AI ช่วยอ่านสรุปการประชุม แล้วแยกแยะคำพูดทั้งหมดออกเป็น 4 ถังโดยอัตโนมัติ เพื่อให้ทีมงานโฟกัสเฉพาะสิ่งที่ "ต้องทำจริงๆ"

1. สิ่งที่ต้องทำรอบนี้ (Current Scope)

AI จะดึงเฉพาะเนื้อหาที่เกี่ยวกับระบบมาเขียนเป็น "สเปก" สำหรับสั่งงาน

2. ไอเดียอนาคต (Future Scope)

เรื่องที่ดี แต่ยังไม่ทำตอนนี้ AI จะจดแยกไว้ เพื่อใช้ตอบกลับ User อย่างนุ่มนวล

3. ข้อมูลที่ไม่ครบ (Missing Info)

จุดเด่นที่สุด! ถ้าตรรกะขาดหาย AI จะไม่มโนเอง แต่จะพ่นคำถามกลับมาให้เราไปถาม User เพิ่ม

4. น้ำ / วาระแทรก (Noise)

คำบ่น เรื่องการเมือง หรือเรื่องที่ไม่เกี่ยวกับระบบ AI จะปัดตกและทิ้งไปเลย

กระบวนการทำงานและเครื่องมือ (Workflow & AI Tools)

เปลี่ยนจากทฤษฎี 4 ถัง สู่สายพานการผลิตจริง โดยเชื่อมต่อ AI Tools ระดับโลกเข้าด้วยกันแบบอัตโนมัติ (Zero-touch Pipeline)

Step 1: Data Capture

อัดเสียงและถอดความ

บอทเข้าร่วมการประชุมเพื่อบันทึกเสียง แยกแยะผู้พูด และถอดเป็นข้อความ (Transcript) 100% ไม่ตกหล่น

🛠️ เครื่องมือที่ใช้: Fireflies.ai / Whisper
Step 2: Logic Engine

สมองกลร่อนตะแกรง

AI นำข้อความมาผ่าน "Prompt กฎเหล็ก" เพื่อแยกวาระลง 4 ถัง และสกัดออกมาเป็น JSON Blueprint

🛠️ เครื่องมือที่ใช้: Claude 3.5 Sonnet
Step 3: Workflow & Board

จัดส่งและสร้างงานลงบอร์ด

รับข้อมูล JSON ไปสร้างการ์ดงานทางเทคนิคบนบอร์ดอัตโนมัติ เพื่อรอให้สถาปนิก (มนุษย์) กดอนุมัติ

🛠️ เครื่องมือที่ใช้: n8n + Jira/Trello

เมื่อผ่านการอนุมัติ... ข้อมูลจะไปไหนต่อ?

JSON Blueprint จะถูกส่งต่อไปเป็น "คำสั่ง" ให้กับ AI Programmer เพื่อช่วยเขียนระบบต่อทันที (Downstream Integration)

JSON Blueprint

นำรายชื่อฟิลด์ไปสร้างตารางเก็บข้อมูลและเขียน SQL Script ทันที

2. เขียนระบบหลังบ้าน

นำ User Story ไปเขียน C# API และ Business Logic ฝั่ง Server

3. วาดหน้าจอเว็บไซต์

สร้างหน้าจอ ปุ่มกด และตารางแสดงผลสำหรับผู้ใช้ (React/UI)

🛠️ ใช้เครื่องมือ

4. ทดสอบหาข้อผิดพลาด

Cursor + Playwright

หน้าตาของ "พิมพ์เขียวดิจิทัล" (The Blueprint)

เพื่อให้เห็นภาพ นี่คือสิ่งที่ BA AI Agent สกัดออกมา (ฟอร์แมต JSON) ซึ่งเป็นมิตรทั้งกับคนและโปรแกรมเมอร์

  "feature_name": "ระบบขออนุมัติส่วนลดพิเศษ",

เป็นมิตรกับทีมงาน

โครงสร้างมีหัวข้อชัดเจน ไม่อ่านยากเหมือนรายงานการประชุมยาวๆ ช่วยให้สถาปนิกตรวจสอบได้ใน 1 นาที

เป็นมิตรกับระบบอัตโนมัติ

สามารถนำไฟล์นี้ส่งให้ n8n อ่าน เพื่อจัดกลุ่มงานและสร้างการ์ดลง Jira / Trello ได้ทันทีโดยไม่ต้องใช้คนพิมพ์

ผลตอบแทนและคุณค่าทางธุรกิจ (Business Value)

ประหยัดเวลาทำเอกสาร (Time-Saving)

ลดระยะเวลาการสรุปการประชุม จาก หลายชั่วโมงเหลือเพียงหลักนาที ทีม BA มีเวลาไปโฟกัสกลยุทธ์มากขึ้น

ลดแรงปะทะ (Friction Reduction)

ใช้ AI เป็น "คนกลาง" ชี้แจงว่าเรื่องไหนอยู่นอกเหนือขอบเขต (Out of scope) ลดการสื่อสารที่ทำให้ผิดใจกันระหว่างแผนก

ลดการแก้งาน (Rework)

สเปกที่ชัดเจน ทำให้ AI Tools (Windsurf/Cursor) ฝั่งโปรแกรมเมอร์ เขียนโค้ดได้แม่นยำ ไม่ต้องรื้อทำใหม่

ก้าวต่อไป (Next Steps)

  • ขอรับฟังความคิดเห็นจากทีมงาน (Feedback)
  • ทดลองนำร่อง (Pilot) กับ 1 โปรเจกต์
  • เชื่อมต่อเข้ากับระบบลงงาน (Backlog Board)
