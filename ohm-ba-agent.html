ยกระดับการเก็บความต้องการของระบบ (Requirement) เปลี่ยนคำพูดทั่วไป ให้กลายเป็น "พิมพ์เขียวดิจิทัล" ที่จัดเรียงโครงสร้างข้อมูลเชิงสัมพันธ์ (Relational Data) อย่างสมบูรณ์

นำเสนอโดย: โอม (Chief Architect)
สถาปัตยกรรม: v4 Enterprise Relational Model

ทำไมเราถึงต้องมีผู้ช่วย AI? (The Pain Points)

ปัญหาคลาสสิกของการพัฒนาระบบ ไม่ใช่การเขียนโค้ด แต่คือ "การสื่อสาร"

ข้อมูลกระจัดกระจาย

เวลาประชุม ผู้ใช้มักเล่าความต้องการผสมกันไปมา ขาดลำดับขั้นตอน ทำให้ BA (คน) ต้องใช้พลังงานมหาศาลในการปะติดปะต่อเรื่องราว

วาระแทรก / นอกเรื่อง

ในคำพูด 100% มักจะมี "เรื่องบ่น" หรือ "ไอเดียอนาคต (ที่ยังไม่ทำตอนนี้)" ปะปนมาด้วย ทำให้เสียเวลาแยกแยะและมักเกิดปัญหาขอบเขตงานบานปลาย

คอขวดที่การทำเอกสาร

ทีมงานต้องเสียเวลาหลายชั่วโมงในการฟังเทปประชุมซ้ำๆ และพิมพ์เอกสาร (Backlog) ให้โปรแกรมเมอร์อ่าน ซึ่งมักจะเกิดการตกหล่น

วิธีแก้ปัญหา: "ตะแกรงร่อนความต้องการด้วย AI"

เราจะใช้ AI ช่วยอ่านสรุปการประชุม แล้วแยกแยะคำพูดทั้งหมดออกเป็น 4 ถังโดยอัตโนมัติ เพื่อให้ทีมงานโฟกัสเฉพาะสิ่งที่ "ต้องทำจริงๆ"

1. สิ่งที่ต้องทำรอบนี้ (Current Scope)

AI จะดึงเฉพาะเนื้อหาที่เกี่ยวกับระบบมาเขียนเป็น "สเปก" พร้อมจัดหมวดหมู่ Type เช่น New Feature (NW), Bug Fix (BG)

2. ไอเดียอนาคต (Future Scope)

เรื่องที่ดี แต่ยังไม่ทำตอนนี้ AI จะจดแยกไว้ระดับ Global Meeting เพื่อใช้ตอบกลับลูกค้าอย่างนุ่มนวล

3. ข้อมูลที่ไม่ครบ (Missing Info)

จุดเด่นที่สุด! ถ้าตรรกะขาดหาย AI จะไม่มโนเอง แต่จะพ่นคำถามกลับมา พร้อมติดป้ายแจ้งเตือนสีเหลือง (Guardrail)

4. น้ำ / วาระแทรก (Noise)

คำบ่น เรื่องการเมืองออฟฟิศ หรือเรื่องส่วนตัว (เช่น เรื่องอาหารแมว) AI จะปัดตกและทิ้งไป 100%

สถาปัตยกรรมข้อมูลแบบเชิงสัมพันธ์ (Relational Data Flow)

อัปเกรดจากระบบ "การ์ดแบนๆ" สู่โครงสร้างระดับ Enterprise ป้องกันปัญหาข้อมูลซ้ำซ้อนและแยกแยะบริบทอย่างชัดเจน

Level 1: Root

Project Master

ข้อมูลระดับบนสุดที่ควบคุมทุกอย่าง เพื่อแยกแยะงานแต่ละโปรเจกต์ออกจากกัน

Level 2: Global Context

Meeting Record

ศูนย์กลางเก็บข้อมูลภาพรวมของการประชุม เช่น Global Future Scope และ Global Missing Info เพื่อไม่ให้ข้อมูลเหล่านี้ไปรกอยู่ในการ์ดงานทุกใบ

Level 3: Execution

Backlog Task

การ์ดงานที่แตกย่อยออกมา พร้อม Row-level AC Type (UI, Logic, Integration) เพื่อชี้เป้าให้โปรแกรมเมอร์เขียนโค้ดได้แม่นยำ

กระบวนการทำงานและเครื่องมือ (Workflow & AI Tools)

จาก Data ดิบ สู่สายพานการผลิตจริง โดยมีมนุษย์เป็นผู้บัญชาการ (Human-in-the-Loop)

Step 1: Data Ingestion

นำเข้าข้อมูล

เลือกโปรเจกต์ ระบุวันที่ แล้วโยน Transcript การประชุมเข้าสู่ระบบ

Step 2: Logic Engine

สมองกลร่อนตะแกรง

ใช้ Gemini 3 Flash Preview สกัดวาระต่างๆ เป็น JSON Blueprint ที่มีโครงสร้างตายตัว 100%

Step 3: Human-in-the-Loop

ตรวจสอบและส่งต่อ

สถาปนิกตรวจสอบการ์ดงานบนบอร์ด เคลียร์ป้าย Missing Info และกดอนุมัติเพื่อส่งให้ SA Agent (Node 03)

เมื่อผ่านการอนุมัติ... ข้อมูลจะไปไหนต่อ?

JSON Blueprint จะถูกส่งต่อไปเป็น "พิมพ์เขียว" ให้กับ AI Programmer เพื่อช่วยเขียนระบบต่อทันที

Validated JSON Contract

🛠️ ผู้รับผิดชอบ

Node 05: Frontend Dev

Node 06: QA Agent

หน้าตาของ "พิมพ์เขียวดิจิทัล" (The v4 Blueprint)

เพื่อให้เห็นภาพ นี่คือสิ่งที่ BA AI Agent สกัดออกมาแบบอัปเดตล่าสุด รองรับ Row-level AC Type และโครงสร้าง Relational Database

blueprint_v4.json 
{
  "feature_name": "ระบบคืนเงินผ่าน E-Wallet",
  "type": "NW", // New Feature
  "user_story": "ในฐานะแอดมิน ฉันต้องการกดปุ่มคืนเงิน...",
  "acceptance_criteria": [
    {
      "type": "Logic",
      "description": "ระบบต้องตัดยอดจาก E-Wallet และโอนคืนให้ลูกค้า"
    }
  ],
  "entities_mentioned": ["e_wallet", "refund_transaction"],
  "task_missing_info": [
    "ระบุ Limit ยอดสูงสุดที่แอดมินสามารถอนุมัติได้เอง"
  ]
}

เจาะจงระดับ Row-Level

Acceptance Criteria ระบุชัดเจนว่าเป็น `UI`, `Logic`, `Validation` หรือ `Integration` เพื่อให้ AI สาย Tech ทำงานต่อได้ตรงจุด

Guardrail 100%

หากมี Missing Info ค้างอยู่ ระบบจะระงับการทำงานปุ่ม Approve ทันที เพื่อป้องกันสเปกขยะหลุดเข้าสู่สายพานการผลิต

ผลตอบแทนและคุณค่าทางธุรกิจ (Business Value)

ประหยัดเวลาทำเอกสาร (Time-Saving)

ลดระยะเวลาการสรุปการประชุม จาก หลายชั่วโมงเหลือเพียงหลักนาที ทีม BA มีเวลาไปโฟกัสกลยุทธ์มากขึ้น

ลดแรงปะทะ (Friction Reduction)

ใช้ AI เป็น "คนกลาง" ชี้แจงว่าเรื่องไหนอยู่นอกเหนือขอบเขต (Out of scope) ลดการสื่อสารที่ทำให้ผิดใจกันระหว่างแผนก

ลดการแก้งาน (Rework)

สเปกที่ชัดเจน ทำให้ AI Tools (Windsurf/Cursor) ฝั่งโปรแกรมเมอร์ เขียนโค้ดได้แม่นยำ ไม่ต้องรื้อทำใหม่

ก้าวต่อไป (Next Steps)

  • ขอรับฟังความคิดเห็นจากทีมงาน (Feedback)
  • นำร่อง (Pilot) พัฒนาระบบด้วย React + Zustand
  • สร้าง API ส่งมอบ JSON ข้ามไปยังฝั่ง SA
