โครงการ Standalone iPad POS | Wellinterparts

โครงสร้างสถาปัตยกรรม 2 ส่วน (Decoupled System)

ระบบถูกออกแบบมาให้ "แยกการทำงานออกจากกันอย่างเด็ดขาด" เพื่อความปลอดภัยและความเสถียรสูงสุด โดยแบ่งออกเป็น 2 แอปพลิเคชันหลัก:

1. iPad POS App

ผู้ใช้งาน: พนักงานขายหน้างาน (Sales)
หน้าที่: แอปพลิเคชันบน iPad สำหรับค้นหาสินค้า รับออเดอร์ ทำงานได้ทั้งโหมดออฟไลน์และออนไลน์

2. POS Admin Portal

ผู้ใช้งาน: ผู้จัดการ และ ไอที (Admin)
หน้าที่: ระบบหลังบ้านบนเบราว์เซอร์ สำหรับจัดการสิทธิ์ พนักงาน ล็อกอุปกรณ์ (Device Binding) และดูประวัติบิลที่รอส่งเข้า ERP

WELLPORTAL (API Gateway)

ทำหน้าที่เป็น "คนกลาง" รับส่งข้อมูลระหว่าง iPad POS, Admin Portal และ ERP System ป้องกันการเจาะระบบเข้าถึงฐานข้อมูลหลักโดยตรง

รายละเอียดหน้าจอ: iPad POS Application

1. Authentication Module

  • 1.1 Login Page: หน้าเข้าระบบ (ตรวจสอบ AD และ Device ID)
  • 1.2 Initial Sync: หน้าจอโหลดข้อมูลพื้นฐานตอนเริ่มวัน

2. POS Operation Module

  • 2.1 Sale Order (Split View):
    • ฝั่งซ้าย: ค้นหาสินค้า (PHILLIPS, DAKEN), กรองหมวดหมู่, เช็ค Onhand
    • ฝั่งขวา: ตะกร้าสินค้า, คำนวณส่วนลดตามเกรดลูกค้า, พักบิล/ยืนยัน
  • 2.2 Customer Selection: ป๊อปอัปค้นหารายชื่อลูกค้า

3. Order History Module

  • 3.1 Dashboard & History:
    • Offline Tab: บิลที่รออินเทอร์เน็ตเพื่อส่งเข้า ERP
    • Online Tab: บิลที่ส่งสำเร็จและได้รับเลข SO กลับมาแล้ว

4. Settings Module

  • 4.1 App Settings: โปรไฟล์พนักงาน (เช่น ปาริชาติ-เปิ้ล)
  • 4.2 Manual Sync: ปุ่มบังคับอัปเดตข้อมูลมาสเตอร์จากระบบ

รายละเอียดหน้าจอ: POS Admin Portal

1. Dashboard

  • 1.1 Main Dashboard: สรุปยอดขายรวมจาก iPad ทุกเครื่อง, ดูสถานะระบบ API

2. Security & Access Module

  • 2.1 User Management: กำหนดสิทธิ์พนักงาน (Sales/Manager)
  • 2.2 Device Binding: ดูสถานะการล็อกเครื่อง iPad
    • ฟังก์ชันสำคัญ: ปุ่ม Reset Device (ปลดล็อกเครื่อง)

3. Transaction Module

  • 3.1 POS Transactions: ตาราง (DataTables) ตรวจสอบบิลขายทั้งหมด ค้นหาตามชื่อเซลส์/วันที่
  • 3.2 Error Retry: กดส่งบิลที่ติด Error กลับเข้า ERP ใหม่

4. Master Data Monitor

  • 4.1 Directory: ตรวจสอบรายชื่อลูกค้า, สินค้า, ตารางราคาที่ดึงมาจาก ERP (View-only)
